概要

  • flask(python3)アプリ

    • k8sでいうところのpodをイメージ
  • やることはTwitter連携と投稿
    • ただのラッパ
    • タグを最初に指定しておけば勝手につけてくれる
  • ドメインもとった
    • gunofma.me
    • 豆の銃
    • ecs.gunofma.me とか eks.gunofma.me とか

  • とりあえずECS(Fargate)で動くところまでできた
    • https://ecs.gunofma.me
    • 勉強会んとき以外は落とす
  • ソースはちょっとリポジトリの分け方とか決まってから…

やりたいこと

  • ECS
    • CI/CD周り
  • EKS
    • EKSでの公開

  • 全般

Cloud Native Kansai#01にいってきました

Kubernetes使ってますか？というのをテーマに、Cloud Nativeなお話を関西で聞ける貴重な会でした。

課題を共感したい

と、思いました。

関西でKubernetesの導入事例がまだまだ少ない中で、個人的に聞きたいのはやはり本番導入の事例、
特に実際使ってみて課題に感じたところやハマったところおよびその対策だったり、
Kubernetesを導入したことで現状の課題が解決されて幸せになったねといったお話。
そういう話がいっぱい出てくることで、導入までのハードルが下がったり、
同じ課題を抱えている人がKubernetes導入してみようかな、という気持ちになったりして、
結果的に関西でもCloud Native界隈が盛り上がるのではないかなぁと。

あとはもうちょっと課題感を共感したい。僕の想像力が足りないだけかもですけど、
課題をさらっと言われてしまうと、何がどう幸せになったのか、
という話があんまりピンと来ない。共感できない。
なので、ふーん、ってなっちゃう。（僕だけ？
たとえば簡単にKubernetes環境ができるのは嬉しいんですけど、
普通にやったらしんどいよね、っていう背景をしっかり共有したほうが
もっと伝わるような気はするんですけどもうそこは暗黙の了解みたいな感じなんですかね。

単純に流行ってるからKubernetesでやろうぜ、っていうのも全然いいんですけど、
別にEC2とRDSとELBでええやん(AWS)、になりがち。適材適所、課題あってこその技術。
そういう意味でちゃんとその点に切り込んでいた森藤さんの発表はよかったと思う。
https://speakerdeck.com/mmorito/cloud-native-kansai-number-01

うだうだ書きましたけど、導入してみましたっていう人やら実際使っててこんな課題ありました、

ところで、JAWS DAYSってフェスじゃね？

タイムテーブルを見てたときに、そう思ったんです。
RADIO CRAZYとか、COUNTDOWN JAPANとか、
でっかいホール借りて、いろんなアーティストが出演して
自分のスキなアーティスト見に行って、刺激を受ける。
JAWS DAYSも、全く同じじゃん、と。
ステージで発表（演奏）する人がいて、それを聴く人がいる。
聴いた人の中には、刺激を受けて自分でもAWSを利用（楽器を演奏）する人も出てくる。
そう考えると、こう置き換えられそう。

  • JAWS DAYSのような大規模イベント = フェス
  • 国内/海外のすごい人を呼んでの講演 = メジャーミュージシャンのライブ
  • 各地域の勉強会 = ライブハウスでのインディーバンドのライブ

そう思えば、会社の中で閉じてるエンジニアは、
大学のサークルの中でしか演奏しないバンドマンのようなもので、
もちろん先輩の演奏見たり、自主練で成長するけれど
やっぱり外でライブするほうが格段に成長スピードが早いわけです。
単純に外のステージに立つということだけでも成長しますし、
いろんなフィードバックももらえます。
僕自身、バンドマンとしてはサークルから飛び出して
外でライブをすることで成長してきましたが、
エンジニアとしては今まで会社に閉じていました。
でも、同じ様にやればいいんだ、ってわかりました。
なんかまぁよくよく考えれば当たり前な話かも知れませんが、
僕にとってはブレイクスルーだったのです。
そうと分かれば、まずはライブハウスでライブができるように頑張ろうかなと思います。
バンドと違うのは１人でやらんとあかんってことかな…
